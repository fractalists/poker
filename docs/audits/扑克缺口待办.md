# Poker Gap Backlog

Date: 2026-04-25

## Scope

本文记录当前德州扑克房间仍欠缺的核心展示、交互与玩法能力。除“当前迭代例外”列出的两项外，以下缺口均明确标记为：短期先不处理，仅作为后续 backlog，不进入当前迭代范围。

## 当前迭代例外

- 展示 / 交互缺口 4：轻量优化桌面筹码动线，让下注、盲注、结算的筹码去向更明确。
- 核心玩法缺口 7：补强规则完整性边界的实现与回归测试，优先覆盖会影响局面合法性的边界。

## 展示 / 交互缺口

### 1. 主池 / 边池展示

Status: 短期先不处理

当前牌桌只展示总 pot 和玩家 in pot，没有把主池、边池、每个玩家可争夺的池子拆开。多人 all-in 后，玩家很难判断“我能赢哪一部分池子”。

### 2. 按 street 分组的完整手牌历史

Status: 短期先不处理

当前 Table feed 是线性事件流，缺少 Preflop / Flop / Turn / River / Showdown 分组、每轮下注汇总和回看入口。后续如果要做 replay 或复盘，需要更完整的事件结构和展示层。

### 3. 摊牌解释与最佳五张牌

Status: 短期先不处理

当前能显示 best hand 名称和赢家，但没有明确高亮最佳五张牌、踢脚牌、平局原因、谁被谁压制。用户在 showdown 时仍需要自己推断。

### 4. 桌面筹码动线

Status: 当前迭代只做轻量优化；更深的筹码动画短期先不处理

当前已有 latest event 和玩家 in pot，但筹码从玩家进入底池、从底池流向赢家的路径不够直观。本轮只补一层轻量 chip flow 提示，完整筹码飞行动画、分池动画、逐街归池动画暂不处理。

### 5. 玩家身份与座位生命周期

Status: 短期先不处理

当前座位能区分人类、AI、busted 等状态，但缺少更完整的 sit in / sit out / leave / rebuy / reconnect 状态提示，也没有稳定的玩家身份视觉层级。

### 6. 移动端深度优化

Status: 短期先不处理

当前移动端已经有可用布局，但还没有针对单手操作、横竖屏切换、长桌滚动、行动区吸附、牌桌缩放做完整体验打磨。

### 7. 房间恢复与连接状态解释

Status: 短期先不处理

当前有 live / reconnecting 状态，但缺少更明确的“正在追上最新牌局”“本地视图落后”“已恢复到第几手”等恢复提示。

## 核心玩法缺口

### 1. 多人真人对局

Status: 短期先不处理

当前重点仍是单人真人 + AI 混合桌。真正多人真人同桌需要座位锁、玩家会话、断线托管、行动超时、权限和状态同步更完整。

### 2. 边池结算产品化

Status: 短期先不处理

底层结算已有边池能力，但产品层还没有把边池结构暴露给前端，也没有对多层 all-in 场景做可读的结算解释。

### 3. 买入、补码、破产与 sit out 流程

Status: 短期先不处理

当前 bankroll 更接近局内模拟状态，缺少正式买入、补码、自动离桌、等待大盲入局、坐下但暂不参局等真实牌桌流程。

### 4. 牌局模式与房间规则配置

Status: 短期先不处理

当前主要是固定 small blind、固定筹码、固定牌桌人数。后续需要支持不同盲注级别、桌型、买入范围、速度、是否允许观战等规则配置。

### 5. AI 决策深度

Status: 短期先不处理

当前 AI 已有混合风格，但还没有完整的下注线记忆、位置感、多人范围推断、诈唬频率管理、对手建模等更强策略。

### 6. 完整 hand replay

Status: 短期先不处理

当前事件流可用于现场更新，但还不是完整可回放协议。后续 replay 需要稳定事件 schema、状态快照、时间轴、逐步播放和复盘视图。

### 7. 规则完整性边界

Status: 当前迭代补强关键边界；完整规则认证短期先不处理

本轮优先覆盖会影响合法行动和对局推进的边界，例如最小加注、all-in 不足额加注、盲注不足额、非参与玩家只能旁观等。完整赌场级规则认证、异常局面恢复和全量规则矩阵暂不处理。
